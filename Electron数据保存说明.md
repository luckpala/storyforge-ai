# StoryForge AI - Electron 数据保存说明

## ✅ 数据保存功能

Electron 版本**完全支持**数据保存到硬盘，这是**内置功能**，无需额外配置。

## 📁 数据保存位置

数据会自动保存在系统标准应用数据目录：

### Windows
```
C:\Users\<用户名>\AppData\Roaming\StoryForge AI\data\
```

### Mac
```
~/Library/Application Support/StoryForge AI/data/
```

### Linux
```
~/.config/StoryForge AI/data/
```

## 📂 保存的数据文件

应用会在数据目录中自动创建以下文件：

1. **`sessions.json`** - 所有会话和故事数据
2. **`settings.json`** - API 配置、应用设置等
3. **`quickPrompts.json`** - 快捷提示词

## 🔧 工作原理

### 数据服务器集成

- **数据服务器已集成在 Electron 主进程中**
- 应用启动时自动启动数据服务器
- 无需手动启动任何服务器
- 数据自动保存到硬盘

### 自动保存机制

- 每次数据修改后，自动保存到硬盘
- 使用防抖机制（2秒延迟），避免频繁写入
- 数据持久化，即使应用关闭也不会丢失

## ✨ 特性

1. **完全自动化**：无需手动操作，数据自动保存
2. **跨浏览器共享**：如果多个浏览器访问，数据会同步（通过数据服务器）
3. **数据持久化**：数据保存在硬盘上，不依赖浏览器缓存
4. **系统标准位置**：使用系统推荐的数据目录，安全可靠

## 🔍 查看数据文件

### Windows
1. 按 `Win + R` 打开运行对话框
2. 输入：`%APPDATA%\StoryForge AI\data`
3. 按回车，即可看到数据文件

### 或者
1. 打开文件资源管理器
2. 在地址栏输入：`C:\Users\<你的用户名>\AppData\Roaming\StoryForge AI\data`
3. 按回车

## ⚠️ 注意事项

1. **数据目录位置**：数据保存在系统应用数据目录，不是项目目录
2. **备份建议**：定期备份 `data` 目录中的文件
3. **卸载应用**：卸载应用时，数据目录不会被删除（可以手动删除）

## 📝 数据迁移

如果需要迁移数据到其他电脑：

1. 复制整个 `data` 目录
2. 在新电脑上安装应用
3. 将复制的 `data` 目录放到新电脑的对应位置
4. 重启应用，数据会自动加载

## 🆚 与 Web 版本的区别

| 特性 | Web 版本 | Electron 版本 |
|------|---------|--------------|
| 数据保存位置 | 项目目录 `data/` | 系统应用数据目录 |
| 需要启动服务器 | 是（需要运行 `data-server.js`） | 否（已集成） |
| 跨浏览器共享 | 需要数据服务器运行 | 自动支持 |
| 数据持久化 | 是 | 是 |
| 卸载后数据 | 保留在项目目录 | 保留在系统目录 |

## ✅ 总结

**Electron 版本完全支持数据保存到硬盘，这是内置功能，无需任何额外配置！**

数据会自动保存，您无需担心数据丢失。
