# C盘垃圾清理工具使用说明

## 📋 脚本说明

提供了两个清理脚本：

### 1. `清理C盘垃圾.bat` - 普通版
- **权限要求**：普通用户权限
- **清理内容**：
  - Windows 临时文件
  - 用户临时文件
  - npm 缓存
  - 回收站
  - 系统日志文件
  - 预读取文件
  - Windows 更新缓存
  - 浏览器缓存（可选）

### 2. `清理C盘垃圾_管理员版.bat` - 管理员版
- **权限要求**：管理员权限（会自动请求）
- **清理内容**：包含普通版所有内容，额外清理：
  - 系统错误报告
  - 旧 Windows 安装文件（Windows.old）
  - 更彻底的 Windows 更新缓存清理
  - 自动运行 Windows 磁盘清理工具

## 🚀 使用方法

### 方法 1：普通版（推荐先试这个）
1. 双击运行 `清理C盘垃圾.bat`
2. 按提示操作
3. 选择是否清理浏览器缓存
4. 等待清理完成

### 方法 2：管理员版（清理更彻底）
1. 双击运行 `清理C盘垃圾_管理员版.bat`
2. 如果提示需要管理员权限，点击"是"
3. 按提示操作
4. 等待清理完成

## ⚠️ 注意事项

### 安全提示
1. **备份重要文件**：清理前请确保重要文件已备份
2. **关闭正在运行的程序**：清理时建议关闭其他程序
3. **不要中断**：清理过程中不要关闭命令行窗口
4. **Windows.old**：删除后无法回退到旧系统，请谨慎选择

### 清理内容说明
- **临时文件**：可以安全删除，系统会自动重新生成
- **npm 缓存**：可以安全删除，下次安装时会重新下载
- **回收站**：删除后无法恢复，请确认没有重要文件
- **浏览器缓存**：清理后需要重新登录网站
- **Windows.old**：旧系统文件，删除后无法回退

## 📊 预期效果

清理后通常可以释放：
- **临时文件**：几百 MB 到几 GB
- **npm 缓存**：几百 MB 到几 GB（取决于使用情况）
- **浏览器缓存**：几百 MB 到几 GB
- **Windows.old**：可能占用 10-30 GB（如果存在）

## 🔧 如果清理后空间仍然不足

### 1. 卸载不需要的程序
- 打开"设置" → "应用" → "应用和功能"
- 卸载不需要的程序

### 2. 移动大文件到其他磁盘
- 检查 `C:\Users\<用户名>\Downloads`
- 检查 `C:\Users\<用户名>\Documents`
- 移动大文件到 D 盘或其他磁盘

### 3. 使用 Windows 存储感知
- 打开"设置" → "系统" → "存储"
- 启用"存储感知"
- 配置自动清理规则

### 4. 检查重复文件
- 使用工具查找重复文件
- 删除不需要的重复文件

### 5. 清理系统还原点
- 打开"系统属性" → "系统保护"
- 点击"配置" → "删除"旧还原点

## 🐛 常见问题

### Q: 脚本运行后没有释放空间？
A: 
1. 检查是否以管理员权限运行（管理员版）
2. 某些文件可能正在被使用，重启后再试
3. 使用 Windows 自带的磁盘清理工具

### Q: 清理后系统变慢？
A: 
1. 这是正常的，系统会重新生成缓存
2. 重启计算机后应该恢复正常
3. 预读取文件会重新生成，不影响性能

### Q: 误删了重要文件？
A: 
1. 检查回收站（如果还没清空）
2. 使用数据恢复工具
3. 从备份恢复

### Q: npm 缓存清理后安装失败？
A: 
1. 重新运行 `npm install`
2. 检查网络连接
3. 清除 npm 缓存后需要重新下载包

## 📝 建议

1. **定期清理**：建议每月清理一次
2. **清理前检查**：清理前检查回收站是否有重要文件
3. **使用存储感知**：启用 Windows 存储感知自动清理
4. **监控磁盘空间**：定期检查磁盘空间使用情况
